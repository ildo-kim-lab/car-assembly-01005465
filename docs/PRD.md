# PRD: 자동차 제조 순서

## 개요

콘솔 메뉴를 통해 사용자가 자동차 부품을 순서대로 선택하여 조립하고, 조립된 차량의 호환성을 검증한 뒤 실행(RUN) 또는 검증(Test)하는 시뮬레이터의 제조 프로세스를 정의한다.

## 제조 순서

자동차는 아래 4단계를 순서대로 진행하며 조립된다. 각 단계는 이전 단계로 되돌아갈 수 있다(`0` 입력).

| 단계 | 항목 | 선택 가능한 부품 |
|---|---|---|
| 1 | 차량 타입 | Sedan, SUV, Truck |
| 2 | 엔진 | GM, TOYOTA, WIA, 고장난 엔진 |
| 3 | 제동장치 | MANDO, CONTINENTAL, BOSCH |
| 4 | 조향장치 | BOSCH, MOBIS |

4단계까지 모두 선택하면 완료 화면으로 이동하여 **RUN**(조립된 차량 동작) 또는 **Test**(조립 유효성 검증)를 선택할 수 있다.

## 조립 호환성 규칙

다음 조합은 조립이 불가능하며, RUN 시 "자동차가 동작되지 않습니다"로 처리되고 Test 시 "FAIL"로 판정된다.

1. 차량 타입이 **Sedan**이면서 제동장치가 **Continental**인 경우
2. 차량 타입이 **SUV**이면서 엔진이 **TOYOTA**인 경우
3. 차량 타입이 **Truck**이면서 엔진이 **WIA**인 경우
4. 차량 타입이 **Truck**이면서 제동장치가 **Mando**인 경우
5. 제동장치가 **Bosch**이면서 조향장치가 **Bosch**가 아닌 경우

위 5가지 규칙에 모두 해당하지 않으면 조립은 유효하며 Test 시 "PASS"로 판정된다.

## RUN 단계 추가 조건

조립이 유효하더라도, 엔진이 **고장난 엔진**인 경우 "엔진이 고장나있습니다. 자동차가 움직이지 않습니다."가 출력되며 차량이 동작하지 않는다.

## 관련 구현

- 제조 순서 진행 및 부품 선택: `assembly.py`의 `main()`, `select_car_type`, `select_engine`, `select_brake`, `select_steering`
- 호환성 검증: `assembly.py`의 `is_valid_check()` (RUN 시 사용), `test_assembly.py`의 `test_produced_car()` (Test 시 사용)
